4. 模型架构

4.1 EmotionCNN — 像素特征提取

这是整个系统最核心的特征提取器。它基于 ResNet18，一个在 ImageNet 上预训练过的经典卷积神经网络。

Input: (B, 3, 224, 224) — 批次 × RGB三通道 × 224像素高 × 224像素宽

ResNet18 Backbone（ImageNet 预训练权重）
├── [冻结] conv1 → bn1 → relu → maxpool     — 底层特征提取器，负责边缘、颜色、纹理等通用视觉特征
├── [可训练] layer1 → layer2                — 中层特征提取器，学习更复杂的局部组合模式
├── [可训练] layer3 → layer4                — 高层特征提取器，学习表情特有的全局模式（嘴角、眉形等）
└── avgpool → (B, 512)                      — 全局平均池化，将空间特征压缩为 512 维向量

Projection Head: Dropout(0.3) → Linear(512, 128) → ReLU → (B, 128)
  • 参数量：总计 ~11.2M，其中 ~10.5M 可训练
  • 冻结策略：只冻结前 4 层（conv1/bn1/relu/maxpool），让 layer1-4 全部可训练。这是因为 ImageNet 的底层特征通用可复用，但表情识别需要学习面部肌肉运动的细微模式，高层必须放开

为什么用 ResNet18 而不是更大的模型？：ResNet18 在精度和速度之间取得了很好的平衡。更大的 ResNet50 参数量是其 4 倍，但在 AffectNet 上精度提升仅 2-3%，推理速度却慢了一倍。对于需要实时检测的场景，ResNet18 是更合适的选择。

4.2 GeoEncoder — 几何特征编码

这是一个 2 层的多层感知机（MLP），负责将 5 个手工设计的几何特征编码为 64 维向量。

Input: (B, 5) — 批次 × 5维几何特征向量
Layer 1: Linear(5, 32) → ReLU   — 将 5 个特征映射到 32 维隐空间
Layer 2: Linear(32, 64) → ReLU  — 进一步映射到 64 维特征空间
Output: (B, 64) — 编码后的几何特征向量

5 个几何特征的含义与表情相关性：

特征 全称 含义 表情相关性
EAR Eye Aspect Ratio 眼睛开合程度（垂直高度/水平宽度） 惊讶时眼睛睁大，EAR 增大
MAR Mouth Aspect Ratio 嘴巴开合程度 惊讶/恐惧时嘴巴张开，MAR 增大
eyebrow_eye_dist 眉眼距离 眉毛中心到眼睛中心的垂直距离 惊讶时眉毛上抬，距离增大
mouth_curvature 嘴角曲率 嘴角相对于嘴中心的偏移 正值=微笑（嘴角上翘），负值=撇嘴
nasolabial_depth 鼻唇沟深度 鼻尖到嘴中心的距离 厌恶时鼻唇沟加深，距离增大

为什么需要几何特征？：纯 CNN 模型只能从像素中学习模式，但表情的本质是面部肌肉运动。这 5 个几何特征直接捕捉了眼睛、嘴巴、眉毛的关键变化，与 CNN 的像素特征互补，构成"像素 + 几何"的多模态融合架构。

4.3 FusionClassifier — 多模态融合分类

融合分类器将像素特征和几何特征拼接在一起，通过全连接层完成最终的 7 类表情分类。

Step 1: Concat — 将 CNN 的 128 维像素特征 + GeoEncoder 的 64 维几何特征拼接为 192 维
Step 2: Linear(192, 128) → ReLU     — 融合特征降维
Step 3: Dropout(0.5)                — 随机丢弃 50% 的神经元，防止过拟合
Step 4: Linear(128, 7)              — 输出 7 类表情的 logits（未归一化的分数）
Step 5: Softmax                     — 将 logits 转换为 7 类概率分布
Output: (B, 7) — 每个样本 7 类表情的概率

为什么用 Dropout(0.5)？：融合层是过拟合的重灾区。192 维的特征直接映射到 7 类，如果没有任何正则化，模型会"死记硬背"训练集。Dropout(0.5) 在训练时随机丢弃一半神经元，强制模型学习更鲁棒的特征组合。

5. 调优历程

整个调优过程经历了 4 个版本，从 46% 的准确率逐步提升到 60% 以上。

v0 — 4 层自定义 CNN（准确率 46%）

最初我让 AI 设计了一个简单的 4 层 CNN（每层 Conv + BN + ReLU + MaxPool），参数量仅 214K。模型从零开始训练，在 AffectNet 33,788 张图片上跑了 2 个 epoch 验证。结果显示准确率只有 46%，F1 只有 0.45——几乎等于随机猜测。

根因：214K 参数对表情识别任务来说严重不足。AffectNet 包含 7 类表情，每类的视觉差异很大，简单的 4 层卷积网络根本没有足够的容量来学习这些复杂的模式。而且从零训练意味着模型需要从头学习边缘、纹理等基础特征，效率极低。

v1 — ResNet18 预训练 backbone（准确率 60%）

我告诉 AI"太低了，优化它"。AI 诊断出问题后，将 EmotionCNN 替换为 ImageNet 预训练的 ResNet18 backbone，仅替换最后的全连接层为 512→128 的投影头。预训练权重带来了 11.2M 参数，其中前 6 层冻结，后 4 层可训练。

效果：准确率从 46% 跃升到 60%，F1 从 0.45 提升到 0.60。这证明了预训练模型的强大——它已经在 ImageNet 的 1400 万张图片上学会了通用的视觉特征，只需要微调就能适配表情识别任务。

v1+geo — 加入预计算几何特征（准确率 ~60%）

虽然准确率没有显著提升，但解决了一个关键的训练/推理一致性问题。训练时，AffectNet 图片没有几何特征（全零输入）；推理时，前端 MediaPipe 发送了真实的几何特征值。这导致 GeoEncoder 的输出分布在训练和推理时完全不同，FusionClassifier 的输入产生偏移。

通过预计算脚本对训练集所有图片运行 MediaPipe，提取真实的几何特征用于训练，保证了训练和推理的输入分布一致。这为后续调优打下了基础。

v2 — 全面调优（目标 63%+）

在 v1+geo 的基础上，我发现了"微笑被误识别为恐惧/厌恶"的问题。AI 给出了 7 项系统性调优：

（1）类别权重平方根平滑

原始权重公式 w_k = N/(K*N_k) 对稀有类（fear/disgust 样本少）补偿过大，权重可达 5-10 倍。这导致模型过度预测这些稀有类，把微笑误判为恐惧。改用 sqrt(N/(K*N_k)) 平方根平滑后，最大权重降到 2-3 倍，稀有类和多数类的预测趋于均衡。

（2）Label Smoothing（标签平滑）

AffectNet 的标签有噪声——同一张脸，有人标为 happy，有人标为 neutral。Label smoothing 将 hard label [0,0,0,1,0,0,0] 变为 soft label [0.014, 0.014, 0.014, 0.914, 0.014, 0.014, 0.014]，让模型不要过度自信，降低对噪声标签的过拟合。

（3）减少冻结层数

v1 中冻结了 ResNet18 前 6 层（conv1, bn1, relu, maxpool, layer1, layer2），只让 layer3 和 layer4 可训练。但 ImageNet 的底层特征（边缘、纹理）通用可复用，中层和高层才是类别特定的。表情识别需要学习面部肌肉运动的细微模式，减少冻结到 4 层后，layer1-4 全部可训练，模型能更好地适配表情特征。

（4）AdamW 优化器 + 分层学习率

v1 使用 Adam 优化器，所有参数同一学习率 0.001。v2 切换到 AdamW（解耦 weight decay），并引入分层学习率：backbone 用 0.1x 学习率（5e-5），head/geo_encoder/classifier 用全学习率（5e-4）。预训练 backbone 已有良好权重，大学习率会破坏已学特征；随机初始化的 head 需要更大学习率快速收敛。

（5）更强的数据增强

v1 只做了水平翻转和 10° 旋转。v2 增加了 RandomErasing（随机遮挡 20% 的图片区域）、RandomAffine（轻微平移 5%）、更强的 ColorJitter（亮度 0.3、对比度 0.3、饱和度 0.2）。这模拟了真实摄像头场景中的各种变化——遮挡、偏移、光照差异，大幅减少了过拟合。

（6）梯度裁剪

新增 clip_grad_norm_(max_norm=5.0)。当梯度超过阈值时自动缩放，防止训练不稳定时的梯度爆炸。对于 ResNet18 这种深层网络，这在训练初期尤为重要。

（7）Dropout 引入 CNN 投影头

在 ResNet18 的 512→128 投影前增加 Dropout(0.3)，进一步防止过拟合。

6. 踩坑记录

模型训练相关

自定义 CNN 准确率仅 46%

这是我犯的第一个错误——让 AI 从零设计一个简单的 CNN 模型。4 层卷积 + BN + MaxPool，214K 参数，在 AffectNet 3 万多张图片上训练。结果出来后我非常失望：准确率 46%，F1 0.45，几乎等于随机猜测。

AI 的诊断很直接：模型容量太小。表情识别是一个复杂的视觉任务，需要理解面部肌肉的细微变化。214K 参数的网络连基本的边缘检测都要从头学起，根本没有余力学习表情特征。解决方案也很明确——用预训练模型。换成 ImageNet 预训练的 ResNet18 后，准确率直接跳到 60%。

微笑被误识别为恐惧/厌恶

这是最棘手的问题。模型训练时看起来指标不错（60% 准确率），但实时检测时发现微笑经常被识别为恐惧或厌恶，happy 类几乎检测不出来。

AI 从三个维度定位了根因：一是类别权重过度补偿，fear/disgust 样本少但权重大了 5-10 倍；二是冻结层太多，模型学不到表情的视觉特征；三是训练和推理的几何特征分布不一致——训练用全零、推理用真实值。这三个问题叠加，导致模型在 happy 类上严重偏向负面表情。修复后，7 项调优一次性应用，happy 的识别率明显提升。

旧权重与新模型不兼容

替换 CNN 架构后，旧的 best_model.pth 无法加载——参数形状不匹配。训练前必须删除旧权重，不使用 --resume 参数从头训练。ResNet18 自带 ImageNet 预训练权重，收敛很快。

前端交互相关

特征点太小看不清

468 个 MediaPipe 面部关键点用绿色小圆点渲染在视频上。默认设置是半径 1px、透明度 0.5，在 640×480 的视频上几乎看不见，用户根本无法分辨特征点的位置。

我告诉 AI"特征点太小看不清"，AI 直接在 CameraView.tsx 的 drawLandmarks 函数中调整了两个参数：半径从 1px 改为 4px，透明度从 0.5 改为 0.9。改完后特征点清晰可见，但仍然保持了绿色，不影响整体视觉效果。

修改前端后页面不变化

改了 .tsx 文件但浏览器刷新后没有变化。原因是 Nginx 服务的是 frontend/dist/ 目录下的构建产物，不是源代码。需要先 npm run build 重新构建，再强制刷新浏览器（Ctrl+Shift+R）清除缓存。

部署相关

摄像头需要 HTTPS

浏览器的 getUserMedia() API 要求安全上下文（HTTPS 或 localhost）。如果通过 HTTP 访问，摄像头权限会被直接拒绝。AutoDL 的 6006 端口自带 HTTPS 支持，通过 Nginx 代理前端和后端，完美解决了这个问题。

9. 总结：AI 编程的核心价值

我做了什么

我做了四件事：提问题、做决策、验收结果、把握方向。

提问题：我不会写 PyTorch 模型，但我知道 46% 的准确率太低了。我不会调 CSS，但我知道特征点看不清。我不会排查分布偏移，但我知道"微笑被识别为恐惧"这个现象不对。问题描述得越具体，AI 给出的解决方案就越精准。

做决策：AI 会给出多个方案供我选择——用 ResNet18 还是 ResNet50？冻结 4 层还是 6 层？用 Adam 还是 AdamW？最终拍板的是我。决策权在我手里，AI 提供的是选项和论据。

验收结果：每个阶段我都会验证——准确率是否提升？推理是否正常？特征点是否可见？不合格就继续迭代。AI 不会自动判断好坏，它需要我的反馈来调整方向。

把握方向：我从一开始就设定了"全栈闭环"的目标——不是只做一个模型，而是做一个完整的产品。这个方向是我定的，AI 沿着这个方向填充了每一层实现。

AI 做了什么

AI 做了三件事：设计架构、编写代码、调试问题。

设计架构：AI 设计了整个系统的架构——前端用 React + TypeScript + MediaPipe，后端用 FastAPI + PyTorch，模型用 ResNet18 + GeoEncoder + FusionClassifier。这不是简单的代码堆砌，而是一个前后端契约一致、数据流清晰的完整架构。

编写代码：从模型定义（ResNet18 backbone 替换 4 层 CNN）、训练管线（TrainingEngine）、推理服务（ModelService）、API 路由（9 个接口），到前端 Hooks（4 个）、页面组件（3 个）、可视化图表（3 个）、数据处理脚本（2 个）、部署脚本（2 个）——全部由 AI 编写。总代码量超过 3000 行，覆盖了数据、模型、训练、推理、前端、部署的完整链路。

调试问题：每一次准确率不达标、推理结果偏差、联调不一致，AI 都能给出根因分析。"微笑被误识别为恐惧"这个问题，AI 从类别权重、冻结层、几何特征分布三个维度定位了根因，并给出了 7 项系统性的修复方案。传统开发中这需要多次沟通、多次验证，AI 通过上下文理解一次性给出了解决方案。

AI 编程的本质

你提出问题和目标，AI 提供从架构到代码到调试的端到端解决方案。

它不是工具，而是合作伙伴——理解你的业务需求、设计系统架构、编写跨层代码、诊断复杂问题。

更重要的是：它让一个人，完成了原本需要 CV 算法工程师 + 后端开发 + 前端开发 + 运维工程师才能完成的工作。
